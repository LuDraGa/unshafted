**Status:** Active
**Last updated:** 2026-04-27
**Baseline branch:** `consolidation`
**Current baseline commit:** `dc17b64 fix(history): align opened report layout`

This document is the durable execution reference for the stabilization and product-clarity work that follows v0.6.7. Use it after context compaction, handoff, or branch switching to understand what has shipped, what remains, and what should happen next.

## Product Direction

Unshafted should become a trustworthy, decision-first contract risk triage tool:

- First-time users should understand the value before technical setup.
- Users should know exactly where contract text, API keys, auth tokens, Drive files, and analysis results go.
- Results should answer the practical signing question first, then expose detail.
- The popup should remain the quick-action surface, not an overloaded full report surface.
- BYOK remains the real scan mechanism through v0.7.0.
- Drive backup is explicit opt-in, not an automatic side effect of sign-in.
- v1 remains upload-first unless current-page analysis is deliberately restored.

## Release Path

### v0.6.8: Stabilization Release

Goal: ship the trust and correctness fixes without expanding product scope.

Include:

- The five baseline commits listed above.
- Any small release-version/package metadata updates required for v0.6.8.
- Release notes focused on security, privacy, stale-result correctness, and verification.

Do not include:

- Billing, hosted credits, full web app, collaboration, new contract templates, or a large report redesign.
- Current-page analysis work unless upload-first positioning is explicitly reversed.

Before release:

- Provider keys that may have existed in `.env` while public builds were produced from vulnerable commits have been rotated externally.
- Confirm Chrome Web Store package is built from the fixed commit.
- Re-run bundle secret scan against the final release artifact.

### v0.7.0: Product Clarity and Reuse

Goal: turn the capable extension into a product normal users can understand, trust, and reuse.

Primary initiatives:

- Strengthen the first-run demo and setup path so users see value before BYOK setup.
- Make the result experience decisively answer: likely okay, negotiate first, pause, or get legal help.
- Mature the history/report loop with reopen, delete, copy/export, and local/Drive status.
- Separate sign-in from Drive backup everywhere in product language and controls.
- Simplify deep-analysis taxonomy into user-task groups.

Do not include:

- Complex Drive document manager.
- Hosted LLM billing or credits.
- Enterprise/team workflows.
- Broad legal workflow platform features.

### v0.8.0: System and Design Maturity

Goal: reduce product/design debt after the v0.7 core loop is settled.

Primary initiatives:

- Consolidate semantic design tokens and shared component variants.
- Mature report/export presentation.
- Resolve upload-only vs current-page analysis positioning and remove dead promises.
- Improve local dev/CI onboarding beyond `.nvmrc` if needed.
- Add migration workflow clarity if Supabase schema continues expanding.

Do not include:

- Lawyer handoff network.
- Negotiation automation marketplace.
- Multi-user workspaces.
- Enterprise admin features.

## Priority Backlog

### P0: Stabilization and Trust

Status: implemented in baseline.

Closed problems:

- Full `.env` injection into extension bundles.
- Provider API-key defaults in client code.
- OAuth token persistence before Supabase auth success.
- Missing OAuth state validation.
- Original source file base64 persistence by default.
- Stale deep-analysis display after re-scan.
- Over-broad structured LLM retry behavior.
- Background message handlers leaving callers unresolved.
- Hidden Drive backup mental model.
- Blocked lint/build/type-check verification.

Remaining P0 release work:

- Prepare v0.6.8 release metadata and notes.
- Confirm final release artifact with bundle scans.

### P1: Product Clarity and Reuse

Status: partially implemented.

Initiatives to execute next:

1. **History and Report Workflow**
   Goal: make saved analyses feel durable and controllable.
   Scope: medium to large.
   Status: implemented, awaiting final manual QA.
   Approach completed: recent analyses show quick/deep reports, copy/export/delete are available, delete and current Drive backup use inline confirmations, local/Drive status is visible, Drive refresh is user-triggered, and opened reports align with the live quick-scan layout.

2. **Decision-First Result Refinement**
   Goal: make users understand the next action before reading details.
   Scope: medium.
   Status: next active workstream.
   Approach: refine the current decision summary into a clearer signing posture, top blockers, top asks, confidence/coverage, and evidence path.

3. **Guided Setup and Demo Polish**
   Goal: make first value happen before technical setup anxiety.
   Scope: medium.
   Approach: improve sample-result entry, setup progress language, provider-key rationale, and handoff between popup and options.

4. **Privacy and Sync Copy Pass**
   Goal: prevent trust collapse from hidden data movement.
   Scope: small to medium.
   Approach: audit popup, options, README, privacy policy, and report copy for consistent statements about LLM provider calls, local keys, Supabase auth, and optional Drive backup.

### P2: System and Scope Cleanup

Status: not started beyond lint baseline.

Initiatives:

1. **Design System Consolidation**
   Goal: prevent UI drift.
   Scope: medium.
   Approach: define semantic tokens for brand, risk, selection, status, focus, and standardize button/input/chip/badge/alert/accordion variants.

2. **Scope Cleanup**
   Goal: make the product easier to explain and maintain.
   Scope: medium.
   Approach: decide whether page analysis is a supported v1 feature. If not, remove or hide user-facing promises and keep code paths clearly secondary.

3. **Dev/CI Onboarding**
   Goal: keep verification repeatable.
   Scope: small.
   Approach: `.nvmrc` is tracked. Consider adding Volta/mise docs or a bootstrap script if contributors keep landing on Node 18.
