# v0.8.0 Execution Checklist

**Status:** Implementation pass complete
**Parent roadmap:** `execution-docs/v0.6.7-to-v0.8.0-roadmap.md`
**Scope:** system maturity, calmer product IA, and positioning cleanup

## Goal

Finish the v0.8.0 maturity phase without adding new product surface area.

v0.8.0 should make Unshafted feel more coherent and maintainable:

- Fewer simultaneous attention targets in the popup.
- More semantic visual language for risk, status, selection, and guidance.
- Cleaner report structure and export presentation.
- Clear upload-first positioning.
- Better local development and migration handoff.

## Non-Goals

- No billing or hosted credits.
- No server-side LLM proxy.
- No web app.
- No lawyer handoff, collaboration, teams, or enterprise admin.
- No current-page analysis restoration unless it is deliberately promoted to first-class product scope.

## Checklist

### 1. Design System Consolidation

- [x] Define shared semantic CSS tokens for brand, surface, risk, status, selection, focus, and text.
- [x] Standardize popup/options button, panel, link-button, alert, badge, and chip semantics where practical.
- [x] Reduce amber overload by reserving it for guidance/brand, not every warning/status/selection state.
- [x] Keep implementation lightweight: CSS tokens/classes before introducing a component library.

### 2. Calmer Popup IA

- [x] Merge scattered privacy/sync and recent-history management into one quieter secondary section.
- [x] Keep the decision card and current primary action visually dominant.
- [x] Make storage/history controls available but less loud by default.
- [x] Preserve all v0.7 functionality: open, delete confirmation, copy/export, Drive refresh, Drive backup prompt, clear-data controls.

### 3. Detailed Report Presentation

- [x] Consolidate deep-analysis sections into user-oriented groups: decision, top risks, negotiation, evidence, other details, caveats.
- [x] Avoid equal-weight accordions for high-impact findings.
- [x] Keep backward compatibility with existing deep-analysis records and Drive history.
- [x] Keep neutral/shareable report language.

### 4. Report Export Maturity

- [x] Improve Markdown export hierarchy so it reads as a report, not a raw dump.
- [x] Include decision, top risks, asks, evidence, coverage, and caveats in a consistent order.
- [x] Keep `.md` export only; no PDF renderer in v0.8.

## Handoff Notes

- The product complaint to address in v0.8 is not lack of features; it is too many attention points.
- Do not solve this by hiding core risk information. Solve it by making management controls secondary.
- The popup should stay the quick-action surface. If report depth still feels too heavy after this pass, the next phase should consider a dedicated report page.
