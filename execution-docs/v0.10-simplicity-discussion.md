# v2 — synthesis after dev feedback

## What changed in my read

Three things you said collapse into one design problem:

- **C** "no scroll, even collapsed feels heavy"
- **D** "Customize is text-heavy and scattered"
- **E** "chips feel separate from the analysis — they're part of it"

You're not asking for smaller cards. You're asking for **one analysis surface** that adapts to the user's question, instead of a stack that grows downward.

The accepted Chrome-popup pattern (NN/G, Primer, evilmartians) is: tiny viewport (~440×600), one CTA, **never scroll the page** — bound the variable region and let it overflow internally if it must. Accordions stack and grow; tabs swap and stay still. Stack is the wrong primitive here.

## North star

> One surface, no page scroll. Each question gets exactly one focal element. Switching question swaps content in place — never pushes the page.

---

## Direction: "Segmented lens" pattern (recommended)

The **verdict** is the anchor. Below it, a **single tab strip** is the only navigation. Tabs *are* the analysis chunks (Blockers / Asks / Evidence / Doc). Active tab content fills the rest of the viewport. The Run-detailed-analysis CTA is anchored to the popup bottom and never scrolls away.

```
┌── header: UNSHAFTED · status · history · upload · avatar ──┐

  Coverage Document.pdf
  Insurance policy summary · 3 parties        ← parties skeleton until LLM returns

┌── Verdict (dark, anchor) ─────────────────────────────────┐
│ LOW                                          RE-SCAN ↻   │
│ Likely okay to proceed                                    │
│ Top concern: Ambiguity in coverage dates                  │
└────────────────────────────────────────────────────────────┘

┌── Lens strip (segmented) ─────────────────────────────────┐
│ ● Blockers · 3   Asks · 3   Evidence   Doc               │
└────────────────────────────────────────────────────────────┘

┌── Lens panel (the only thing that changes) ───────────────┐
│ Ambiguity in coverage dates                  [HIGH]       │
│ Cover start/end dates not clearly aligned…                │
│                                                            │
│ Incomplete plan details                       [MEDIUM]     │
│ Some plan fields aren't fully clear…                      │
│                                                            │
│ Unclear trip dates vs travel status           [MEDIUM]     │
│ Nature of Trip is listed but exact travel…                │
└────────────────────────────────────────────────────────────┘

┌── Sticky CTA bar (always at bottom) ──────────────────────┐
│ Reviewed as Traveler · Pricing, Term ⚙   Run analysis →  │
└────────────────────────────────────────────────────────────┘
```

## Interaction model

**Lens strip:**

- Single-select. Default = `Blockers` (the answer).
- Active = filled pill; inactive = ghost pill with count + severity dot if any high.
- Click another tab → panel content swaps in-place. Page height does not change.
- Keyboard: `←/→` between lenses (proper `role="tablist"`).

**Lens panel:**

- Bounded height. Internal `overflow-y: auto` only if content exceeds.
- Page itself never scrolls. The CTA bar never moves.

**Verdict card:**

- Compressed: pill + headline + ONE-LINE top-concern preview pulled from worst blocker.
- That preview is what makes "Top blockers collapsed" not feel like hiding the answer — the answer is already on screen.

**CTA bar (the run-deep panel):**

- Always visible, anchored bottom. Single horizontal row.
- Left side = current scope summary ("Reviewed as Traveler · Pricing, Term"). Tap the ⚙ → scope sheet slides up over the lens panel (does NOT push the page). Two rows of chips (role / priorities). Done button collapses the sheet.
- Right side = primary action button.
- Once deep is running → bar shows progress; once deep returns → bar vanishes (or becomes "Re-scan").

**Doc strip (top):**

- Filename instantly (from PDF parse).
- Doc type instantly (rough heuristic) — or after quick-scan if LLM-derived.
- Parties: muted skeleton "· — parties" until quick-scan returns; then fades in to "· 3 parties". Tap parties text → opens the `Doc` lens, which holds the full party list + extraction details.
- This is the only thing on the page that changes shape during loading; it does so in an unobtrusive footer-of-header position.

## Why this beats Chrome-extension neighbors

Most extension popups either (a) stack to scroll or (b) jump to a new full page. The lens pattern gives you (c) — a single popup view that *behaves* like a full page through internal navigation. NN/G's "Tabs, Used Right" specifically endorses tabs for *content of similar nature, of which only one is needed at a time*. Blockers / Asks / Evidence / Doc are all "reasons the verdict is what it is" — same nature, one at a time. Exactly the case for tabs.

## Alternatives I considered (and rejected)

- **Bottom-sheet stack** (verdict on top, swipe-up sheet for blockers/asks): too iOS-native; gesture discoverability is bad in a Chrome popup with mouse.
- **Carousel** (left/right pages for Blockers/Asks/Evidence): forces sequence, loses random access, motion fatigue.
- **Two-column** (verdict left, blockers right): 440px is too narrow.
