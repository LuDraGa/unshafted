# v0.7 History and Report Workflow

**Status:** Implemented, awaiting final manual QA
**Parent roadmap:** `execution-docs/v0.6.7-to-v0.8.0-roadmap.md`
**Scope:** v0.7 product clarity and reuse

## Goal

Make saved analyses feel durable, understandable, and controllable without turning the popup into a full document manager.

Users should be able to:

- See recent reports with clear risk and save state.
- See quick scans in history immediately, not only detailed analyses.
- Reopen a report without losing the current scan unexpectedly.
- Copy or export a report in a stakeholder-friendly format.
- Delete a report with clear local and Drive behavior.
- Understand whether a report is local-only, Drive backup was requested, or it was restored from Drive.
- Manually refresh Drive-backed reports from the history panel.
- Opening a recent report should scroll/focus the loaded report so the state change is obvious.
- Enabling Drive while an analysis is visible should ask whether to back up that current analysis too.

## Non-Goals

- No full Drive file browser.
- No multi-report comparison.
- No collaboration or sharing links.
- No hosted report web page.
- No new billing, credits, or server-side report storage.

## Current State

Implemented:

- History exists in `chrome.storage.local`.
- Quick scans and deep analyses are pushed to history.
- History can be reopened.
- Reports can be copied and exported as Markdown.
- Records can be deleted after inline confirmation; local deletion removes matching rows by id and content hash, and Drive deletion is attempted by content hash.
- Drive hydration can recreate history when local history is empty.
- History can be manually refreshed from Drive while the panel is open.
- Save/sync state is stored per report as `Local only`, `Drive backup requested`, or `Restored from Drive`.
- Signed-out users see sign-in guidance instead of a dead Drive toggle.
- Enabling Drive with a current analysis visible shows an in-popup backup prompt.
- Opened reports scroll/focus visibly and use the same quick-scan structure as live upload results.

## Proposed Behavior

### History List

Each report row should show:

- Document name.
- Risk level.
- Created date.
- Whether deep analysis exists.
- Save state:
  - `Local only`
  - `Drive backup requested`
  - `Restored from Drive`

Actions:

- `Open`
- `Copy report`
- `Export .md`
- `Delete`
- `Refresh Drive` when signed in

Signed-out users see sign-in guidance instead of an unusable Drive enable toggle.

Enabling Drive does not silently upload the visible report. It shows an in-popup prompt first, then marks the local row as `Drive backup requested` and requests quick/deep sync when available.

### Opened Report

When a history report is open:

- Show a compact report toolbar above the result.
- Include Back, Copy report, Export `.md`, Delete.
- Use neutral, shareable labels.
- Render quick-scan content using the same top-level structure as the live upload flow: verdict strip, coverage, summary, parties, flags, obligations, topics, and warnings.
- For detailed reports, show the same quick-scan context before the detailed-analysis section.

### Export Format

Use Markdown for v0.7:

- Easy to copy, save, email, or paste into Docs.
- No new PDF/rendering dependency.
- Works inside extension constraints.

Report sections:

- Document and reviewed role.
- Risk posture.
- Summary / bottom line.
- Top quick-scan flags.
- Top negotiation asks.
- Suggested edits.
- Questions to ask.
- Caveats and disclaimer.

## Data Model

Add optional `storageState` to `HistoryRecordSchema`:

- `local-only`
- `drive-backup-requested`
- `restored-from-drive`

Backward compatibility:

- Existing records default to `local-only`.
- Drive hydration marks records as `restored-from-drive`.
- Quick scan records are saved immediately.
- Detailed analysis replaces the same report row when it completes.
- New deep analysis records created while Drive backup is enabled use `drive-backup-requested`.

Important wording:

- Do not say `Backed up to Drive` for newly synced reports because sync is fire-and-forget and not confirmed in the UI.

Manual checks:

- With a deep analysis in history, Open/Copy/Export/Delete actions are visible.
- Export downloads a `.md` file with neutral report copy.
- Copy writes the same report body to clipboard.
- Drive-restored records show `Restored from Drive`.
- Existing history records parse as `Local only`.
- Quick-scan-only results appear in history before detailed analysis.
- Running detailed analysis updates the existing quick-scan row rather than adding a duplicate.
- Clicking `Refresh Drive` can hydrate synced reports after the history panel is already open.
- Opening a report scrolls to and focuses the opened report toolbar.
- Delete removes matching local rows by id and content hash after confirmation.
- Signed-out state does not show `Enable Drive`.
- Enabling Drive with a current quick/deep result prompts to back up the visible analysis.
- Opened quick-scan-only reports show useful content, not just metadata.
- Opened reports visually match the live quick-scan IA while keeping saved-report actions read-only.
