# v0.7 Privacy and Sync Copy

**Status:** Implementation pass 1 complete
**Parent roadmap:** `execution-docs/v0.6.7-to-v0.8.0-roadmap.md`
**Scope:** v0.7 trust and product clarity

## Goal

Make data movement understandable at the decision points where users choose to scan, sign in, or enable Drive backup.

The product should consistently say:

- API keys stay local in extension storage.
- Contract text goes to the selected AI provider when analysis runs.
- Supabase stores auth/profile data only, not contract text or reports.
- Reports save locally by default.
- Drive backup is explicit opt-in for signed-in users.
- New scans with Drive backup enabled can upload source files and report JSON.
- Backing up an already-visible local report requires confirmation and may only back up report JSON if the original source file is no longer in memory.

## Non-Goals

- No legal-policy rewrite beyond product accuracy.
- No new consent modal.
- No telemetry, analytics, or backend report storage.
- No change to OAuth scopes in this copy pass.

## Implemented Behavior

- Popup privacy/sync copy states that reports save locally and contract text goes to the selected provider when scanning.
- Options page includes a data-flow note below provider setup.
- README now points to the v0.8 roadmap and no longer describes old Phase 2/3 sequencing as current work.
- README Drive sync wording now distinguishes requested vs confirmed backup state.
- Privacy policy now reflects sign-in-gated deep analysis and explicit Drive backup behavior.
- Privacy policy clarifies that old local-only source files may not be backfilled.
