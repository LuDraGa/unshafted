# v0.7 Guided Setup and Demo

**Status:** Implementation pass 1 complete
**Parent roadmap:** `execution-docs/v0.6.7-to-v0.8.0-roadmap.md`
**Scope:** v0.7 product clarity and first-run usability

## Goal

Let a new user understand Unshafted's value before they configure provider infrastructure.

The first-run path should make these facts obvious:

- The sample analysis works without an API key.
- Real scans for the user's own contracts require a provider key for now.
- The key is stored locally.
- Contract text goes to the selected provider only when scanning.
- Setup is a short prerequisite, not the product's main task.

## Non-Goals

- No hosted credits or billing.
- No server-side LLM proxy.
- No new onboarding account system.
- No full visual redesign of the popup.

## Implemented Behavior

- No-key state uses `Set up API key` as the primary action instead of a disabled upload button.
- `View sample analysis` remains available without setup.
- Popup onboarding copy explains the setup path and why the provider key matters.
- Options onboarding copy uses step language for provider, key, save, and test.
- Options page includes a data-flow note covering local key storage, provider-bound contract text, and separate Drive backup.
