# v0.7 Attention Simplification

**Status:** Implementation pass 1 complete; manual QA passed
**Parent roadmap:** `execution-docs/v0.6.7-to-v0.8.0-roadmap.md`
**Scope:** v0.7 product clarity and reuse

## Goal

Reduce the number of simultaneous attention targets in the popup while preserving the same utility.

The product should feel like:

1. First, answer what the user should do.
2. Then show the minimum supporting context.
3. Then expose controls, history, storage, and raw details as secondary actions.

## Non-Goals

- No full visual redesign in this pass.
- No new design system tokens yet; that belongs in v0.8.
- No dedicated report page yet.
- No removal of history, Drive, role customization, or export behavior.

## Implemented Pass 1

- The decision card is now the result summary anchor.
- Duplicate quick-scan summary accordions were removed from live and opened reports.
- Onboarding/result guidance now targets the decision card instead of a separate summary accordion.
- Document metadata no longer competes above the result after quick scan exists; it moves below the core result as `Document details`.

## Remaining Noise Sources

- Storage/privacy and Recent analyses still appear as separate management blocks below the main result.
- Deep-analysis details still expose many categories and should be consolidated into evidence/details/caveats.
- The popup header, sticky footer, upload icon, auth avatar, guide, options, storage, and history can still create too many controls at once.
- Visual tokens are still mood-based rather than semantic; v0.8 should address this systematically.

User feedback after pass 1: behavior works, but the popup still has too many simultaneous attention points and does not feel calm enough. Treat the next visual/IA simplification as v0.8 work, not as more v0.7 stabilization churn.

## Suggested Next Passes

1. Combine storage/privacy and history into one quieter `Library and sync` management section.
2. Simplify deep-analysis accordions into `Evidence`, `Negotiation details`, `Other details`, and `Caveats`.
3. Audit header/footer actions so only the current primary action is visually loud.
4. Move report-heavy experiences to a dedicated extension page if popup density remains high.
